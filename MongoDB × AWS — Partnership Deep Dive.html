The Partnership Story

From database to cloud-native platform — built together

MongoDB and AWS have co-evolved since the earliest days of cloud computing. What began as a natural fit — a developer-first database on the world's most popular cloud — has matured into one of the deepest, most strategically aligned ISV partnerships in the industry.

MongoDB Atlas
🍃

The unified data platform

MongoDB Atlas is the developer data platform — combining operational data, vector embeddings, full-text search, and stream processing in a single, consistent experience. On AWS, Atlas runs across 25+ regions, giving teams the global reach to serve customers everywhere while staying close to their compute.

Amazon Web Services
☁️

The cloud where builders build

AWS powers over 30% of the internet's infrastructure and has invested more than $100B in cloud and AI capacity. As an AWS Advanced Technology Partner and ISV Accelerate member, MongoDB works alongside AWS field teams to co-sell Atlas as the data layer of choice for enterprise AI and application development.

Technical Integration Depth

Built into the AWS fabric — not just hosted on it

MongoDB Atlas isn't simply deployed on AWS infrastructure — it's deeply wired into the AWS service ecosystem, enabling builders to compose sophisticated applications without leaving their cloud-native toolchain.

🔒

AWS PrivateLink

Private, dedicated network connectivity between Atlas clusters and AWS VPCs — no public internet exposure, no complex peering rules. Enterprise-grade isolation that satisfies the most demanding security and compliance requirements.

⚡

AWS Lambda Triggers

Atlas Database Triggers fire Lambda functions in response to real-time database events — inserts, updates, deletes. Build event-driven architectures where data changes automatically activate downstream processing, notifications, or analytics pipelines.

🔄

Amazon EventBridge

Publish MongoDB Atlas events directly to EventBridge, connecting database changes to a serverless event bus that fans out to dozens of AWS services. Decouple your data layer from application logic and scale each component independently.

🤖

Amazon Bedrock + Vector Search

Atlas Vector Search stores and retrieves embeddings generated by Amazon Bedrock foundation models, enabling production-ready RAG (Retrieval-Augmented Generation) applications. Build semantic search, recommendation engines, and AI copilots directly on your operational data.

🧠

Amazon SageMaker

Feed operational MongoDB data directly into SageMaker training pipelines and inference endpoints. Keep ML models in sync with fresh production data without complex ETL layers, and write predictions back to Atlas for real-time application consumption.

🔧

AWS CloudFormation

Provision and manage MongoDB Atlas clusters as infrastructure-as-code alongside your AWS resources. One CloudFormation template defines your entire stack — EC2, Lambda, API Gateway, and Atlas — enabling repeatable, version-controlled deployments at every scale.

Customer Journey

From first line of code to global scale

The MongoDB × AWS partnership is designed around developer velocity and enterprise confidence — meeting teams at every stage of their journey.

01

Start free, scale instantly

Developers spin up a free MongoDB Atlas cluster on AWS in under three minutes. Atlas handles provisioning, replication, backups, and patching — teams write application code, not infrastructure config. When traffic spikes, Atlas scales horizontally without downtime.

02

Transact through AWS Marketplace

When teams are ready to commit, they purchase MongoDB Atlas directly on AWS Marketplace, applying pre-committed EDP or MACC spend. Finance teams are happy — the spend hits the cloud line item. Procurement takes days, not months. Deals close 40% faster than traditional direct procurement.

03

Activate co-sell for enterprise

For enterprise accounts, MongoDB and AWS field teams engage jointly — bringing solution architects, co-sell credits, and shared customer success resources. ISVs co-selling with AWS see win rates jump from 22% to 47%. The AWS relationship provides enterprise customers the assurance of a supported, validated deployment.

04

Build AI applications on a unified data layer

With Atlas Vector Search and deep integrations into Amazon Bedrock and SageMaker, teams add AI capabilities directly on top of their existing operational data. No separate vector database, no data duplication — semantic search, RAG applications, and real-time recommendations all run from the same Atlas cluster that powers the core application.

05

Go global across 25+ AWS regions

Atlas Global Clusters distribute data across AWS regions with configurable zone sharding — serving users from the region closest to them while meeting data residency requirements. MongoDB's global footprint on AWS gives enterprises the reach to build truly worldwide applications from day one.
